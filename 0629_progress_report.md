# 进度汇报(本阶段)

本阶段工作聚焦于自进化智能驾驶算法在 CARLA 环境下自车控制异常的排查与修复,重点攻克了车辆行驶过程中持续向单侧横向偏移、最终驶入对向车道的问题。为剥离既有实验性改动对诊断的干扰,本阶段在一个全新隔离的 `interfuser_origin` 干净基线目录下开展验证,以逐帧驾驶日志作为主要诊断手段,系统性地定位了横向控制环节的开环缺陷,并通过引入闭环车道居中反馈,使车辆恢复稳定沿车道中线行驶的能力。

在基线构建与问题定位方面,基于 opendilab 原始 InterFuser 搭建了独立的干净运行环境(独立 conda 环境与路由、控制器配置),并以此复现并确认了自车的核心异常:车辆在行驶过程中持续向道路单侧偏移,偏移量随时间系统性累积,最终偏出车道乃至驶入对向车道。经逐帧数据分析,该偏移呈现出明确的方向一致性(车道偏移均值稳定在 +0.47m 一侧),据此判定其并非随机扰动,而是横向控制环节存在系统性缺陷。

单侧偏移问题的修复是本阶段的核心成果。经代码追踪确认,系统虽已具备基于仿真器地图真值的 `lane_offset` 检测能力(可精确给出自车相对车道中线的横向偏移量),但该信号默认未纳入控制反馈,横向位置完全跟随模型预测航点开环运行;加之模型在当前车辆配置下所预测的航点本身存在系统性侧偏,二者叠加导致车辆持续向同一侧漂移。针对此,通过启用基于 `lane_offset` 的闭环车道居中反馈(设定合理增益),将实时横向偏差引入转向控制。实测车道偏移均值由修复前的 +0.47m(持续偏向一侧)收敛至 −0.21m(贴近车道中线),偏移量上限被压缩至不再越过车道中线,且转向控制全程未出现高频正负翻转,即未引入任何新增摆动。修复完成后,车辆能够沿既定路由稳定向前行驶,巡航速度维持在 4–5 m/s,过弯与停起行为正常,单侧偏移及驶入对向车道的现象彻底消除。

当前系统的核心遗留挑战主要集中在背景交通的行为异常。测试表明,背景车辆普遍处于停滞状态(速度近零),且有较高比例位于行车道之外;经多组对照实验,已排除该现象由服务器过载或 Traffic Manager 物理模式配置不当所致,初步定位为交通管理模块的生成与控制逻辑在当前仿真版本下未能有效驱动背景车辆。此外,评测流程偶发的段错误与世界加载卡顿属仿真稳定性问题,目前以重试机制规避。背景交通行为的彻底修复将作为后续工作的关键卡点继续推进解决。
